# CoC 跑团助手 · 使用指南

本文档详细介绍 CoC 跑团助手的各项功能与操作流程。安装与配置请参见 [README.md](README.md)。

---

## 一、界面概览

界面分为**左侧栏**与**主区域**两部分。

### 左侧栏

- **标题栏**：应用名称、EN/中 语言切换按钮、设置按钮 ⚙
- **人物卡**：人物卡列表、新建/生成、导入
- **骰子**：手动投骰输入框（如 `1d100`、`3d6×5`）及投骰按钮
- **KP Prompt 设置**：自定义守秘人系统提示词
- **剧本**：已上传剧本列表、上传 PDF 剧本
- **地图**：已上传地图图片列表、上传地图图片

### 主区域

- **KP 模式**：选择对话模式、人物卡、输入框，与 AI KP 对话
- **行动日志**：对话、投骰、技能检定记录，支持刷新、导出、新会话
- **人物卡详情**：点击左侧人物卡后显示（可编辑）

---

## 二、首次使用流程

1. **打开设置**：点击右上角齿轮 ⚙，配置 LLM 提供商和 API Key（详见下方「设置与配置」）
2. **上传剧本**：在左侧「剧本」处点击「上传 PDF 剧本」，选择 CoC 模组 PDF
3. **准备人物卡**：新建或导入人物卡
4. **开始对话**：选择剧本与模式，在输入框输入行动或提问，点击「发送」

---

## 三、设置与配置

点击右上角 **⚙ 设置** 打开设置弹窗。

**推荐：** 使用阿里百炼的 **qwen3.5-plus** 模型可获得最佳跑团体验。若选择其他 LLM，效果可能有所差异。

| 项目 | 说明 |
|------|------|
| **LLM 提供商** | 选择 AI 服务商（百炼、DeepSeek、Kimi、智谱等），或 Ollama/LM Studio（本地） |
| **API 地址** | 仅在选择「自定义」时显示，用于填写兼容 OpenAI 的 API 地址 |
| **模型** | 对应服务商的模型名称（如 qwen3.5-plus、deepseek-chat） |
| **API Key** | 大部分云端服务商需填写；Ollama、LM Studio 本地无需填写 |
| **语言** | 界面语言：中文 / English |

保存后立即生效，配置保存在 `data/api_config.json`。

---

## 四、剧本管理

### 上传剧本

1. 点击「**上传 PDF 剧本**」
2. 选择 CoC 模组或剧本的 PDF 文件
3. 上传成功后，剧本会出现在左侧剧本列表中，并显示字数

### 选择剧本

- 点击左侧剧本列表中的某一项，使其高亮
- 未选择剧本时无法发送对话，会提示「请先上传剧本」

### 剧本作用

上传的剧本会经过解析与索引，KP 在回复时会根据当前对话内容检索相关段落，作为剧情与判定的参考。

---

## 五、人物卡管理

### 新建人物卡

1. 点击「**+ 新建/生成**」
2. 填写表单（基本信息、属性、技能、战斗、人设等）
3. **投属性**：点击「投属性」自动投 3D6 生成 STR、CON 等
4. **AI 辅助分配技能**：选择职业后点击「AI 辅助分配」，由 AI 按职业生成技能
5. **AI 生成人设**：填写姓名、年龄、职业、国籍后点击「AI 生成人设」
6. 点击「**保存**」

### 导入人物卡

- 点击「**导入**」，支持格式：PDF、JSON、TXT、MD、HTML、DOCX
- 导入后人物卡会出现在列表中

### 编辑人物卡

- 在左侧点击人物卡，右侧显示详情
- 点击「**编辑**」进入编辑表单，修改后保存

### 删除人物卡

- 在左侧人物卡项右侧点击「删」按钮
- 确认后删除

---

## 六、地图模块

上传地图图片（PNG/JPG/WEBP/BMP），建议使用规范命名（如「1-博物馆正门.png」）。每次与 KP 对话时：1) 将**全部地图文件名**注入 prompt，由 KP 判断是否适用；2) 若当前语境（玩家输入 + 剧本片段）中出现某地图文件名中的关键词，且 LLM 支持多模态，会传入该地图图像供识别；3) **强制 KP 在回复开头**完成：

1. **判断是否适用某张地图**：若有则指明「适用地图：文件名」，若无则说明「无适用地图」
2. **复述玩家/角色的请求**
3. **指出可能存在的疑问或歧义**

适合《奈亚拉托提普的面具》等超长篇剧本，33 张规范命名的地图即可。

---

## 七、KP 模式与对话

### 对话模式

| 模式 | 说明 |
|------|------|
| **指定人物卡行动** | 以所选人物卡的身份进行行动描述，KP 会结合人物卡信息回复 |
| **玩家询问 KP** | 以玩家身份向 KP 提问（规则、剧情等），不绑定人物卡 |

### 开始对话

1. 确保已选择剧本（左侧剧本列表高亮）
2. 选择模式：人物卡行动 或 玩家询问 KP
3. 若为人物卡行动，在「选择人物卡」下拉框中选择人物
4. 在输入框输入行动描述或提问，点击「**发送**」或按 Enter（Shift+Enter 换行）

### AI 推断行动

- 在「指定人物卡行动」模式下，选择人物卡后，可点击「**AI 推断行动**」
- AI 会根据当前剧情和人物卡，给出建议的行动文案，可直接填入输入框或在此基础上修改

---

## 八、投骰系统

### 对话中的自动投骰

KP 在回复中可以使用特殊标记，系统会自动执行投骰并替换为结果：

| 标记 | 说明 | 示例 |
|------|------|------|
| `[ROLL:技能名\|技能值]` | 技能检定（D100） | `[ROLL:侦查|60]` |
| `[DICE:表达式]` | 任意骰子表达式 | `[DICE:1d100]`、`[DICE:3d6×5]` |

-  skill 与 value 用 `|` 分隔
- 支持常见表达式：`1d100`、`3d6`、`2d6+6`、`3d6×5` 等

### 手动投骰（左侧栏）

- 用于 Luck、伤害等非技能检定
- 在「骰子」输入框中输入表达式（如 `1d100`、`3d6×5`），点击「**投骰**」
- 结果会显示在输入框下方，并写入行动日志

---

## 九、行动日志

- **刷新**：重新加载当前会话日志
- **导出 TXT**：将日志导出为文本文件，便于保存或复盘
- **新会话**：清空当前会话并开始新的一局（对话历史会清空，人物卡和剧本保留）

日志内容包括：玩家/角色发言、KP 回复、投骰结果、技能检定及结果。

---

## 十、KP Prompt 设置

点击「**KP Prompt 设置**」可自定义守秘人的系统提示词。

- 用于设定 KP 风格、规则偏好、剧本使用方式等
- 支持在提示词中说明 `[ROLL]`、`[DICE]` 的用法
- 点击「**恢复默认**」可还原为内置默认提示词

---

## 十一、典型跑团流程

1. 打开应用，确保设置中 API Key 已配置
2. 上传本次使用的模组 PDF
3. 新建或导入调查员人物卡
4. 选择剧本，选择「指定人物卡行动」并选中人物卡
5. 输入调查员的行动（如「我检查桌上的信件」），发送
6. KP 根据剧本和行动回复剧情，需要检定时会在回复中使用 `[ROLL:技能|值]`
7. 系统自动投骰，KP 回复中显示检定结果
8. 继续对话推进剧情；需要时可使用「新会话」开始新的一局

---

## 十二、常见操作提示

- **无法发送消息**：检查是否已选择剧本
- **API 错误**：检查设置中的 API Key 和模型是否填写正确
- **投骰无效**：确认 KP 回复中使用了正确的 `[ROLL:...]` 或 `[DICE:...]` 格式
- **想切换语言**：点击右上角 EN/中 或 设置 → 语言
